# 📑 코드리뷰 가이드라인

코드리뷰를 통해 내가 아닌 다른 사람의 시선으로 피드백을 받는 것은 빠른 성장을 위해 중요한 과정입니다. 코드리뷰를 통해 우리는 버그를 미리 발견할 수도 있고, 내가 미처 인지하지 못한 중복 코드를 발견하고 모듈의 재사용성을 증대할 수 있습니다. 무엇보다도 다른 사람이 읽고 이해할 수 있는, 즉 협업할 수 있는 개발을 할 수 있다는 가장 큰 장점이 있습니다. 이렇게 장점이 많은 코드리뷰! 들어서 좋은건 알지만, 시작할 때 막막하다면 아래의 리스트를 참고해주세요 😃

# ✅ 코드리뷰 할 때 확인해볼 사항

  • 1. 기능 요구 사항이 정상적으로 동작되는가
  • 2. 버그는 없는가
  • 3. 코드가 더 간단히 작성될 수 있는가
  • 4. 중복을 줄일 수 있는가
  • 5. 네이밍이 적절한지
  • 6. 배울만한 점, 내가 몰랐던 점
  • 7. 칭찬

# 💬 코드리뷰할 때 좋은 커뮤니케이션 방법

# 🙋🏻‍♂️대답을 강요하는 것이 아닌 질문 또는 권유를 해주세요.

ex) 'onHandler'라는 네이밍이 정확히 어떤 역할을 할 수 있는지 고민되는 것 같아요. 'onClickHandler'와 같이 구체적인 네이밍으로 바꿔보는건 어떨까요?
